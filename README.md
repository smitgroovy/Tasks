# Tasks

A collection of web development projects generated from structured prompts.

## Prompts

Every prompt used to generate this repository is documented below.

### Prompt 1 — Task1: Personal Portfolio

> i have created repo at github which is https://github.com/smitgroovy/Tasks.git , now create one folder named Task1 inside create files for my personal details and all details take from [PDF 1]. it must be responsive and must me user interactive.

**Purpose:** Generate a responsive, interactive personal portfolio website inside `Task1/`, populated with personal details extracted from the user's PDF resume.

### Prompt 2 — Task1: Design Refinement

> Make the UI feel handcrafted by a senior product designer, with exceptional spacing, typography, hierarchy, and attention to detail. Avoid common AI-generated design patterns, template-like layouts, and trendy effects unless they serve a clear purpose.

**Purpose:** Elevate the visual quality of the portfolio UI — professional-grade spacing, typography, and hierarchy while avoiding generic AI-generated aesthetics.

### Prompt 3 — Task2: Flowdo Initial Build

> Task2: Flowdo - Full-stack todo app with planning docs, backend API, and React frontend

**Purpose:** Generate a full-stack task management application (Flowdo) inside `Task2/` with planning docs, Express/MongoDB backend, and React/Vite/TypeScript frontend. Produced the complete initial codebase including product vision, PRD, database schema, API design, and architecture docs alongside the working app.

### Prompt 4 — Task2: Remove Authentication

> Remove authentication - single user mode, no login required

**Purpose:** Strip out all auth-related code (JWT, login/register pages, auth middleware, user model) to simplify the app to single-user mode.

### Prompt 5 — Task2: Meaningful Categories

> Meaningful categories - defaults, task counts in sidebar, category detail view, category picker in task form

**Purpose:** Enhance the categories feature with default categories, task count badges in the sidebar, a dedicated category detail page, and a category picker in the task creation form.

### Prompt 6 — Task2: Task Expand on Click

> Task expand on click - shows description, due date, created time, completion time

**Purpose:** Make task cards expandable on click to reveal description, due date, created time, and completion time inline.

### Prompt 7 — Task3: Prompt Engineering Bootcamp

> Complete the Prompt Engineering Bootcamp: attend workshop (5 anti-patterns), Exercise 1 (rewrite 10 bad prompts), Exercise 2 (5 prompt styles), Exercise 3 (few-shot prompting), build prompt-library.md with 10 reusable templates.

**Purpose:** Master prompt engineering fundamentals through structured exercises and build a reusable prompt library for future tasks.
